# 04 硬件-固件行为映射

## 目的

将公开资料中的“硬件结构、交互行为、故障现象”映射为可执行的二开关注点，减少后续调试盲区。

## 行为链路总览

1. PD 协议诱骗成功 -> 获得 20V 供电。
2. 电源/降压链路稳定 -> MCU + OLED 正常上电。
3. 温度采样稳定 -> PID 控制输出 PWM。
4. 导电结构稳定（探针或铜柱方案） -> 加热板持续工作。

依据：[S1][S2]

## 对后续二开的模块分解建议

- `power-bringup`: 先验证 20V 拉起与稳压链路。
- `thermal-sensing`: 聚焦 PT1000 采样稳定性与异常值处理。
- `ui-input`: 复核双键交互时序与状态切换延迟。
- `mechanical-conduction`: 验证导电结构在热循环下的可靠性。
